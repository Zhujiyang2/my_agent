# Agent Loop 设计

## 一句话

Agent 是一个"大模型调用工具、工具返回结果、大模型再决策"的循环，直到大模型觉得任务完成了才停下来。

## 三句话说清楚

1. **用户发消息 → 大模型思考 → 决定调用哪些工具**：每条用户消息进入上下文后，大模型判断需要执行什么操作（查文件、跑命令、读写记忆等），生成工具调用指令。
2. **工具执行 → 结果写回上下文 → 大模型再次决策**：工具执行完后，结果追加到对话历史里，大模型看到新信息后决定是继续调工具还是生成最终回复。
3. **循环有上限保护，上下文快满了会自动压缩**：最多循环 N 轮防止死循环，连续失败会提前终止；上下文超过预算时，旧的工具输出会被摘要压缩，保证不超出大模型的输入限制。

## 关键组件

```
用户输入
  → [ContextManager: 组装历史消息]
    → [chatStream: 调用大模型 API]
      → 大模型返回 tool_calls
        → [ToolRegistry: 查找并执行工具]
          → 结果追加到 ContextManager
            → 回到 chatStream（下一轮）
              → 大模型返回最终文本 → 输出
```

## 核心文件

- `src/agent/loop.ts` — 主循环，组装上下文、调大模型、执行工具、处理异步任务回调
- `src/tools/registry.ts` — 工具注册表，所有可用工具的动态查找
- `src/tools/executor.ts` — 工具执行器，含高危命令安全确认
- `bin/my-agent.ts` — 入口，初始化所有组件并启动 REPL
