# 昇腾NPU单机环境预检

当用户请求检查或查看 NPU 环境状态（如"检查NPU环境""查看NPU状态""NPU环境怎么样""昇腾环境预检"等类似语义）时，使用本技能。

## 执行流程

1. **运行检查脚本**：使用 `run_command` 执行与本技能同目录下的 `check.sh`：
   ```
   bash skills/ascend-npu-single-check/check.sh
   ```
   该脚本自动完成以下所有检查：
   - 驱动与固件版本（npu-smi 可用性、Driver 版本、Firmware 版本）
   - 逐芯片硬件状态（健康状态、温度、功耗、AI Core 使用率、内存使用量）
   - 网络通信（P2P 连通矩阵、RoCE/IB 网卡状态）

2. **展示报告**：脚本输出终端友好文本报告和 JSON 报告。直接向用户展示文本部分即可。

3. **解读关键发现**：
   - 若 `npu-smi` 不可用 → 提示用户检查 CANN 是否安装、环境变量 `ASCEND_HOME` 是否已配置
   - 若有芯片健康状态非 OK（显示 FAIL） → 建议检查芯片物理连接、尝试重启 NPU 或联系硬件支持
   - 若有芯片温度或内存数值较高 → **仅如实展示原始数值，不做 PASS/FAIL 判断**，由用户或后续 Agent 自行评估
   - 若 P2P 连通性异常 → 提示检查 NPU 互联拓扑和芯片在位状态
   - 若 RoCE 网卡不存在（显示 N/A） → 说明 RDMA 网卡为可选配置，单机训练不依赖 RDMA

4. **报告保存位置**：脚本自动将 JSON 报告保存到 `./output/npu_check_<时间戳>.json`，向用户告知保存路径。

## 注意事项

- 温度和内存使用率**不做 PASS/FAIL 判定**，仅报告数值供人工分析
- 当前仅适用于**单机场景**，不检查多机 HCCL 跨节点通信（后续分布式检查会扩展）
- 本技能聚焦昇腾 NPU 环境，不涉及 GPU/NVIDIA 相关内容
